Backup & Restore - Help Screen

Overview

The Backup & Restore screen allows you to create database backups of all Proclaim data and restore from previous backups. This is essential for migrating to a new server, creating safety snapshots before major changes, or recovering from data loss.

Important: Database backups do NOT include media files (images, audio, video). You must backup the /media/com_proclaim/ folder separately using FTP or your hosting file manager.

Export (Backup)

Create a backup of your Proclaim database tables.

Export Options

Button Description
Download Backup Creates a backup and immediately downloads the SQL file to your computer. Best for creating off-site backups or transferring to another server.
Save Backup to Server Creates a backup and saves it to /media/com_proclaim/backup/ on the server. Useful for quick snapshots before making changes.

Saved Backups

Previously saved backups are listed in the dropdown. These are stored in /media/com_proclaim/backup/ and can be used for import or downloaded later.

Tip: Backup filenames include the site name, date, and Proclaim version (e.g., proclaim-backup_mysite_2024-01-15_v10.1.0.sql) for easy identification.

Import (Restore)

Restore Proclaim data from a backup file. You have three options for specifying the backup source:

Import Options

Method Description
Upload Backup File Upload a .sql or .zip backup file from your computer. Limited by your server's upload size settings.
Import from Backup Folder Select a previously saved backup from the server's backup folder.
Advanced: Tmp Folder For large files, upload via FTP to the Joomla tmp folder and specify the path here.
Large Files: If your backup exceeds the upload limit shown, use FTP to upload the file to your server's tmp folder, then use the Advanced Options to specify the file path.

Migration Guide

To migrate Proclaim to a new server:

  1. Export - Download a backup from your old server
  2. Install - Install Proclaim on the new server
  3. Import - Upload and restore the backup file
  4. Media Files - Copy /media/com_proclaim/ via FTP
  5. Verify - Check the Control Panel for any post-migration messages
Version Compatibility: Backups from Proclaim 9.x and later will be automatically upgraded to the current version during import. Database migrations run automatically.

What's Included in Backups

Included (Database)

  • Messages (sermons/studies)
  • Teachers
  • Series
  • Topics
  • Locations
  • Media file records
  • Servers
  • Podcasts
  • Templates and template codes
  • Comments
  • All configuration settings

NOT Included (Must backup separately)

  • Uploaded images
  • Audio files
  • Video files
  • Thumbnail images
  • Any files in /media/com_proclaim/

Troubleshooting

Import Fails or Times Out

  • For large databases, increase PHP max_execution_time in php.ini
  • Use the FTP upload method for large backup files
  • Check available disk space on the server

Upload Size Limit

The maximum upload size is determined by your PHP settings:

  • upload_max_filesize - Maximum file upload size
  • post_max_size - Maximum POST data size (should be larger)

Missing Media After Import

Media files are not included in database backups. Copy the entire /media/com_proclaim/ folder from your old server to the new one.
