Archive Messages - Help Screen

Overview

The Archive tool allows you to bulk archive (unpublish) older messages based on their age. This is useful for managing large message libraries by hiding older content from public view while keeping it available in the backend for reference.

Note: Archiving does NOT delete messages. Archived messages are set to "unpublished" state and can be republished at any time from the Messages list.

How to Use

Step 1: Set the Timeframe

Enter a number in the timeframe field. This determines how old a message must be before it gets archived.

Step 2: Choose the Interval

Select whether the timeframe is measured in Years, Months, or Days:

Interval Example
Years Timeframe of "2" archives messages older than 2 years
Months Timeframe of "6" archives messages older than 6 months
Days Timeframe of "90" archives messages older than 90 days

Step 3: Archive

Click the Archive button. You will be asked to confirm the action before any messages are modified.

Tip: The archive operation shows a progress indicator and reports how many messages were archived when complete.

What Gets Archived

Only published messages with a study date older than the specified timeframe will be archived. Messages that are already unpublished, trashed, or archived are not affected.

Restoring Archived Messages

To restore (republish) archived messages:

  1. Go to Messages in the Proclaim menu
  2. Use the Status filter to show "Unpublished" messages
  3. Select the messages you want to restore
  4. Click the Publish button in the toolbar

Use Cases

  • Website Performance: Reduce the number of published messages to speed up frontend queries
  • Content Management: Keep your public message list focused on recent content
  • Seasonal Cleanup: Archive old series or dated content periodically
Before Archiving: Consider creating a backup first if you're archiving a large number of messages. While archived messages can be restored individually, having a backup provides an extra safety net.
