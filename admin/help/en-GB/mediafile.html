Media File Entry - Help Screen

Overview

The Media File Entry form allows you to create and edit individual media file records. Each media file is attached to a sermon/study and can include audio recordings, video files, PDF documents, or links to external media services like YouTube.

The form dynamically changes based on the server type you select, showing relevant fields for that particular media source.

Toolbar Actions

Button Description
Save Saves the media file and returns to the Media Files list.
Save & Close Saves the media file and closes the edit form.
Save & New Saves the current media file and opens a blank form to create another.
Save as Copy Creates a duplicate of the current media file with a new ID. Useful for creating similar entries.
Cancel Discards any unsaved changes and returns to the Media Files list.
Help Opens this help screen.

General Tab

Core Fields

Field Description
Study (Required) Select the sermon or study this media file belongs to. Click the Select button to open a modal and search for studies.
Create Date The date associated with this media file. Defaults to the study date if created from a study.
Server (Required) Select the server type that hosts this media. The form will update to show server-specific fields. See Server Types below.
Podcast Assign this media file to one or more podcasts. Hold Ctrl/Cmd to select multiple podcasts.
ID The unique database identifier. Auto-generated for new records.
Status Publication status: Published, Unpublished, Archived, or Trashed.
Access The access level required to view this media file (Public, Registered, etc.).
Language Language assignment for multilingual sites. Use "All" for all languages.
Comment Optional comment that appears under the media file or as a tooltip, depending on template settings.

Server Types

The server type determines where the media file is hosted and what configuration options are available. When you change the server, the form reloads with appropriate fields for that server type.

Local / Legacy Server

For files stored directly on your web server.

Field Description
Filename (Required) Select or upload a media file from your server. Files are stored in /images/biblestudy/media/ by default.
File Size The size of the media file. Can be entered manually or calculated automatically.
Duration Media duration in Hours, Minutes, and Seconds format.

YouTube Server

For embedding YouTube videos.

Field Description
YouTube Video ID The unique identifier from the YouTube URL (e.g., for youtube.com/watch?v=ABC123, enter ABC123).
Duration Video duration in Hours, Minutes, and Seconds format.
Tip: You can find the YouTube video ID in the URL after v= or in the share link after youtu.be/.

Media Parameters

These settings control how the media file is displayed and behaves on the frontend.

Link and Player Settings

Field Description
Link Target How links open: New Window (_blank), Same Window (_self), Parent Frame (_parent), or Full Window (_top). Use Global inherits from server settings.
Download Icon Controls download link display: No icon, Show download icon, or Show only download icon.
Player How media plays: Direct Link (browser default), HTML5 Player (embedded player), or Embed Code.
Popup For HTML5 Player: Popup window, Inline on page, or Lightbox (modal overlay).

Display Settings

Field Description
Media Image Custom image/thumbnail for this media file.
Button/Icon Display How to display the media link: None, Button Only, Button and Icon, or Icon Only.
Button Text Text displayed on the button (e.g., "Audio", "Video", "Download").
Button Type Bootstrap button style: Primary (blue), Success (green), Info (cyan), Warning (yellow), Danger (red), or No Color.
Button Color Custom color override for the button.
Icon Type Select from predefined icons or use a custom icon.
Custom Icon CSS class for a custom icon (e.g., Font Awesome class).
Icon Size Font size for the icon in pixels.

Advanced Settings

Field Description
MIME Type The media file's MIME type (e.g., audio/mpeg, video/mp4). Usually auto-detected.
Player Width Custom width for the embedded player in pixels.
Player Height Custom height for the embedded player in pixels.
Popup Title Custom title text displayed in the popup header.
Popup Footer Custom text displayed in the popup footer.
Popup Margin Margin around the popup content in pixels.
Autostart Whether the media should start playing automatically when loaded.
Note: Many browsers block autostart for audio/video. This setting may not work in all cases due to browser policies.

Publishing Tab

Standard Joomla publishing information:

Field Description
Created Date and time the record was created (read-only).
Created By User who created the record (read-only).
Created By Alias Optional alias for the creator name.
Modified Date and time of last modification (read-only).
Modified By User who last modified the record (read-only).

Permissions Tab

Configure access control for this specific media file. You can set permissions for different user groups to control who can edit, delete, or change the state of this media file.

Available permissions:

  • Delete - Allow users to delete this media file
  • Edit - Allow users to edit this media file
  • Edit State - Allow users to change the publication state

Size Converter Tool

A utility tool is available to help convert file sizes between KB, MB, and GB. This is useful when manually entering file sizes.

Best Practices

  • Always select the correct server type before entering media details
  • Use descriptive comments to help users understand what each media file contains
  • Assign media files to podcasts if you want them included in your podcast feeds
  • Set appropriate access levels if some media should be restricted to registered users
  • Enter accurate duration times to help users know how long the media is
  • Use consistent button text and styling across similar media types
