Server Entry - Help Screen

Overview

The Server Entry form allows you to configure media servers. Servers define where media files are hosted and how they should be accessed. Different server types have different configuration options.

Toolbar Actions

Button Description
Save Saves the server and remains on the edit form.
Save & Close Saves the server and returns to the Servers list.
Cancel Discards any unsaved changes and returns to the list.
Help Opens this help screen.

Details Tab

Basic Information

Field Description
Server Name (Required) A descriptive name for the server configuration.
Server Type The type of media server. Options include Local, Amazon S3, YouTube, Vimeo, etc.
Status Publication status.
Access Access level for this server configuration.

Server Types

Local Server

For media files hosted on your own web server.

Field Description
Server Path Base path to media files (e.g., /images/biblestudy/media/)
Image Path Path to server-specific images.

YouTube

For embedding YouTube videos.

Field Description
Default Settings Default player settings for YouTube embeds.

Vimeo

For embedding Vimeo videos.

Amazon S3

For media hosted on Amazon S3 buckets.

Field Description
Bucket Name Your S3 bucket name.
AWS Region The AWS region where your bucket is located.
Access Key / Secret AWS credentials for accessing private buckets.
Security Note: Keep AWS credentials secure. Consider using IAM roles with minimal permissions.

Player Settings

Field Description
Default Player How media should play by default (Direct Link, HTML5 Player, etc.)
Player Width/Height Default dimensions for embedded players.
Autostart Whether media should start playing automatically.
Tip: Create separate server configurations for different media sources to keep settings organized.
